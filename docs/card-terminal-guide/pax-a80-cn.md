### PAX A80 基本操作说明
> 本操作指南也适用于 A920 Pro，操作流程一致。 
#### 1. Sale（付款）

用于正常刷卡或电子支付交易。

**操作步骤：**

1. 在主屏幕点击 **Sale**
2. 输入交易金额
3. 选择支付方式：
   - Tap / Apple Pay（非接触支付）
   - Chip（插入芯片卡）
   - Swipe（刷磁条）
4. 按屏幕提示完成操作
5. 客人签名（如需要）
6. 交易成功后打印或发送小票

#### 2. Void Sale（取消付款）

用于**当天交易且尚未进行 Batch（结账）之前**的取消操作。

**操作步骤：**

1. 在主屏幕点击 **Void Sale**
2. 输入管理员密码（默认是1或当前日期比如今天10月15号2025年，则为10/15/2025）
3. 选择需要取消的交易  
   （可从交易记录中选择或输入交易编号）
4. 确认取消
5. 系统提示 Void 成功

> ⚠️ 已完成 Batch 的交易无法 Void，只能使用 Refund。

#### 3. Add Tips（添加小费）

用于**交易已完成但尚未 Batch** 的订单补录小费。

**操作步骤：**

1. 在主屏幕点击 **Func**
2. 选择 **Tip Menu**
3. 选择对应的交易记录
4. 输入小费金额
5. 确认提交
6. 小费将在 Batch 时一并结算

> 常用于客人签字后填写小费的情况。

#### 4. Batch（结账 / 结算）

用于将当天所有已完成的交易**正式提交给银行结算**。

**操作步骤：**

1. 在主屏幕点击 **Func**
2. 选择 **Batch**
3. 确认 **Batch to Close**
4. 等待系统处理完成
5. 打印 Batch 报表（建议保留）

> ⚠️ 建议每天营业结束后至少进行一次 Batch。

#### 5. Force Auth（强制授权 / 强制添加小费）

用于在**已获得银行授权码（Authorization Code）**的情况下完成交易，  
多用于网络异常或特殊补录场景。

**操作步骤：**

1. 在主屏幕点击 **Force Auth**
2. 输入交易金额或小费金额
3. 按提示输入卡号、有效期
4. 输入小票上的 **Auth Code**
5. 确认交易完成

> ⚠️ 此功能风险较高，仅在明确有可能会被客人 Dispute 的风险后使用。

#### 6. Refund（退款）

用于**已完成 Batch 的交易退款**。

**操作步骤：**

1. 在主屏幕点击 **Return**
2. 输入管理员密码（默认是1或当前日期比如今天10月15号2025年，则为10/15/2025）
3. 输入退款金额
4. 让客人刷卡 / 插卡 / Tap / 输入卡号信息
5. 确认退款成功
6. 打印退款小票

> 退款到账时间通常为 **3–7 个工作日**，具体以客人银行为准。
