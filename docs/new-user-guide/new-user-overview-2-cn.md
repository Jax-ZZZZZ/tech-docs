### 支付方式安全与风险

为保障交易安全并降低拒付（Chargeback）风险，  
建议优先使用以下支付方式（安全性由高到低）：

1. **Tap / Contactless / Apple Pay**
2. **Chip（芯片卡）**
3. **Swipe（磁条卡）**
4. **Manual Entry（手动输入卡号）**

说明：
- Tap 与 Chip 的安全等级相同，手续费通常一致
- Tap / Chip 交易：
  - 客人通常无法以“盗刷”为理由发起部分类型的拒付
  - 但仍可能存在其他拒付情形
- **相比手动输入卡号（Manual Entry），Tap / Chip 等更安全的支付方式，  
  商户服务费（Merchant Service Fee）通常也会略低一些**
- 不同拒付类型需要提供不同的举证材料

---

### 手动输入卡号（Manual Entry）风险提示

手动输入卡号通常仅需：
- 卡号
- 有效期

因此该方式风险较高，系统默认启用以下安全措施：
- **强制要求输入 CVV 安全码**

⚠️ 即便满足以上条件，银行在拒付处理中仍然**优先保护持卡人**。  
如发生拒付，需同时满足以下条件，才有可能赢回款项：

- 客人签字
- CVV 匹配
- 邮编（ZIP Code）匹配
- 账单地址门牌号匹配

✅ **强烈建议：**
- 针对大金额订单
- 尤其是手动输入卡号的交易  
务必要求查看实体信用卡，必要时核对驾照。
